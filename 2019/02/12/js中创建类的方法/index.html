js中创建类的方法

js也是面向对象编程的语言，但是在es6之前并没有Class。只知道js的对象或者类都是函数，函数是一等公民嘛。

首先，必须搞清楚一个一直困扰的问题，类和对象到底有什么区别，概念上说的很清楚，类就是对象的一个模板。典型的面向对象编程语言（比如C++和Java），存在“类”（class）这样一个概念。所谓“类”就是对象的模板，对象就是“类”的实例。JavaScript语言没有“类”，而改用构造函数（constructor）作为对象的模板。

此时觉得js类和对象真是“暧昧”。

js中创建对象直接new Object(),再添加属性方法。显然，这个Object可以是个“元类”。

下面总结js中创建类（对象）的方法：

1.工厂方式：

//通过工厂方式创建对象，先定义一个工厂方法
function createObj(){
    var obj  =  new Object();
    obj.name="xxx";
    obj.say=function(){
        alert("我是xxx");
    }
    return obj;
}
//调用工厂方法创建对象:
var obj1  =  createObj();


//也可以用这种形式
function createObj(){
    var obj  =  {};  //这样生成对象
    obj.name="xxx";
    obj.say=function(){
        alert("我是xxx");
    }
    return obj;
}
var obj1  =  createObj();

这种方式的问题是每一次通过工厂方法去创建一个对象，这个对象的属性name和方法say都必须重新创建一次，浪费内存。而且没有解决对象识别问题（即怎样知道一个对象的类型）。

2.构造函数：

//创建一个构造器，构造函数首字母大写
function Obj(){
    this.name="xxx";
    this.say=function(){
        alert("我是" + this.name);
    };
}
//利用构造器，通过new关键字生成对象
var obj1=new Obj();
var obj2=new Obj();

这个 是js类的基本方式，构造函数名字以大写首字母开头，这是借鉴其他OO语言，主要是为了区别ECMAScript中其他的函数。

这个解决了对象识别问题：


    alert(obj1.constructor ==  Obj)  // true
    alert(obj2.constructor ==  Obj)  // true


    alert(obj1 instanceof ==  Object)  // true
    alert(obj1.constructor ==  Obj)  // true
    alert(obj2 instanceof ==  Object)  // true
    alert(obj2.constructor ==  Obj)  // true

但是，和工厂模式一样，每个方法都要再实例上重新创建一遍。

3.原型方法：

//用空函数创建一个类
function Obj(){
}
//在类的原型链上添加属性和方法
Obj.prototype.name="xxx";
Obj.prototype.say=function(){
    alert("我是" + this.name);
}
//生成对象
var obj1=new Obj();

这个方式的解决了每个实例重新创建方法的问题。缺点是，当有引用属性时，改变一个对象的这个属性也会改变其他对象的这个属性。因为一个引用属性，都是指向的同一个地方,原型链上属性是所有实例共享的。

4.原型/构造联合方式

//用构造函数定义对象的非函数属性
function Obj(name){
    this.name=name;
    this.arr=new Array('a','b');
}
//用原型方式定义对象的方法
Obj.prototype.say=function(){
    alert("我是" + this.name);
}
//生成对象
var obj1 = new Obj('xxx');

这种是目前用的最多的创建类和对象的方式，将方法和属性根据自己需要用不同的方式封装。

5.动态原型方式

//动态原型方式和原型/构造混合方式的原理相似，唯一的区别就是赋予对象方法的位置
function Person(name, sex) {
    this.name = name;
    this.sex = sex;
    if (typeof this.say != "function") {
        Person.prototype.say = function () {
            alert(this.name);
        }
    }
}
var man =new Person ("凯撒", "男");
man.say();//凯撒

动态原型模式是将所有的信息都封装到构造函数中，构造函数中，只用say不存在的情况下，才会将它添加到原型中。这段代码只有在初次调用时才会执行。
